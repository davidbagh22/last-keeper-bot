# Telegram-бот «Последний хранитель»

Отдельный Telegram-бот иммерсивного культурно-просветительского проекта «Последний хранитель». Репозиторий и инфраструктура не связаны с ботом ОО «ЭРА».

## Концепция

Дом Москвы в Ереване превращается в повреждённый Архив памяти России. Участники проходят четыре реальные локации, выполняют офлайн-задания, получают коды и принимают командные решения. Бот выступает цифровым Архивариусом: сопровождает маршрут, проводит личные игровые испытания, фиксирует решения и собирает их в финальную механику «эффекта бабочки».

Даты программы: **16–17 ноября 2026 года**. Основная аудитория: молодые российские соотечественники 16–26 лет.

## Возможности участника

- атмосферная регистрация без лишних полей;
- равномерное распределение по пяти командам;
- программа дня и персональный маршрут;
- навигация к следующей точке;
- четыре личных мини-испытания внутри Telegram;
- четыре артефакта личного прогресса;
- капитанский ввод кодов после офлайн-заданий;
- два содержательных командных выбора на каждой локации;
- скрытые параметры и пять финальных архетипов Архива;
- вопросы организаторам с ответом прямо в Telegram.

## Админ-панель

Команда `/admin` доступна только суперадминистраторам из `SUPERADMIN_IDS` и администраторам, которых суперадминистратор добавил из самой панели.

В панели доступны:

- показатели регистрации и прохождения;
- список команд и участников;
- назначение капитанов из списка команды;
- обращения участников и ответы;
- рассылка с подтверждением;
- пауза и возобновление приёма кодов;
- открытие общего финала;
- экспорт участников, обращений и JSON-summary для финального видео;
- добавление и удаление администраторов;
- журнал ключевых действий.

Обычный администратор не может назначать других администраторов. Суперадминистратор не может быть удалён через Telegram.

## Локации

1. **Код культуры** — русский язык, литература, культурные символы и передача смысла.
2. **Цена открытия** — российская наука, метод, проверка и ответственность.
3. **Выбор эпохи** — происхождение источников и сложная историческая память.
4. **Голоса памяти** — различие документа и человеческого свидетельства.
5. **Эффект бабочки** — общий финал, который собирает последствия решений.

## Команды

- `/start` — регистрация или возвращение в Архив;
- `/program` — программа проекта;
- `/progress` — личный и командный прогресс;
- `/help` — объяснение механики;
- `/whoami` — Telegram ID и текущие роли;
- `/admin` — панель Архивариуса;
- `/reply НОМЕР текст` — ответ на обращение;
- `/cancel` — отмена текущего действия;
- `/resetme` — удалить собственную тестовую регистрацию, доступно администраторам.

## Коды локаций по умолчанию

- «Код культуры» — `CULT26`;
- «Цена открытия» — `SCI26`;
- «Выбор эпохи» — `HIST26`;
- «Голоса памяти» — `MEM26`.

Перед мероприятием коды следует заменить в переменных окружения Render.

## Развёртывание на Render

1. Создайте Blueprint из репозитория.
2. Укажите секрет `BOT_TOKEN`.
3. Проверьте `SUPERADMIN_IDS` — это Telegram ID человека, который управляет доступом к админке.
4. Дождитесь статуса `Live`.
5. Откройте `/health`: ожидается `status: ok` и username бота.
6. Отправьте `/start`.

Webhook автоматически восстанавливается при каждом запуске. Случайные пробелы в токене удаляются, а webhook-секрет преобразуется в разрешённый Telegram формат.

## Локальный запуск

```bash
cp .env.example .env
# заполните BOT_TOKEN и при необходимости PUBLIC_BASE_URL
pip install -r requirements.txt
uvicorn main:web --reload
```

## Проверка

```bash
python -m compileall -q app.py main.py config.py game_data.py storage.py
python -m unittest discover -s tests -v
```

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `BOT_TOKEN` | токен отдельного Telegram-бота |
| `SUPERADMIN_IDS` | Telegram ID суперадминистраторов через запятую |
| `WEBHOOK_SECRET` | секрет Telegram webhook |
| `DATABASE_PATH` | путь к SQLite |
| `EVENT_DATES` | даты потоков через запятую |
| `TEAM_CAPACITY` | вместимость одной команды |
| `LOCATION_CODE_*` | коды игровых локаций |
| `PUBLIC_BASE_URL` | внешний адрес при локальном/альтернативном хостинге |

## Важное ограничение текущего хостинга

В бесплатной конфигурации Render SQLite находится в `/tmp`. При полном пересоздании сервиса или новом деплое тестовые данные могут быть удалены. Перед открытием регистрации для 150 реальных участников необходимо подключить постоянное хранилище или PostgreSQL. Игровая логика и миграции уже отделены от текстов, поэтому перенос базы можно выполнить без изменения пользовательских сценариев.
