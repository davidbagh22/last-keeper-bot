# Telegram-бот «Последний хранитель»

Отдельный Telegram-бот иммерсивного проекта «Последний хранитель». Репозиторий и инфраструктура не связаны с ботом ОО «ЭРА».

## Реализовано

- регистрация участника;
- выбор дня участия: 16 или 17 ноября 2026 года;
- равномерное распределение по пяти командам;
- персональные маршруты;
- капитанские коды четырёх локаций;
- четыре командных решения и скрытые параметры;
- пять финальных версий Архива;
- раздел «Мои фрагменты»;
- обращения к организаторам;
- админ-панель, назначение капитана и CSV-экспорт;
- webhook-запуск на Render;
- локальный polling-запуск.

## Команды

- `/start` — регистрация или главное меню;
- `/whoami` — показать Telegram ID и роль;
- `/resetme` — удалить свою тестовую регистрацию;
- `/admin` — панель администратора;
- `/captain TELEGRAM_ID` — назначить капитана.

## Коды локаций по умолчанию

- «Код культуры» — `CULT26`;
- «Цена открытия» — `SCI26`;
- «Выбор эпохи» — `HIST26`;
- «Голоса памяти» — `MEM26`.

Перед мероприятием коды следует заменить в переменных окружения Render.

## Развёртывание на Render

1. Создайте новый Blueprint из этого репозитория.
2. При создании сервиса укажите секрет `BOT_TOKEN`.
3. После успешного деплоя откройте `/health`.
4. Ответ должен содержать `"status": "ok"` и username Telegram-бота.
5. Отправьте боту `/start`.

Render автоматически передаёт сервису внешний адрес через `RENDER_EXTERNAL_URL`; бот устанавливает отдельный webhook при запуске.

## Локальный запуск

```bash
cp .env.example .env
# заполните BOT_TOKEN
pip install -r requirements.txt
python app.py
```

## Переменные окружения

| Переменная | Назначение |
|---|---|
| `BOT_TOKEN` | токен отдельного Telegram-бота |
| `ADMIN_IDS` | Telegram ID администраторов через запятую |
| `WEBHOOK_SECRET` | секрет webhook |
| `DATABASE_PATH` | путь к SQLite |
| `EVENT_DATES` | даты через запятую |
| `TEAM_CAPACITY` | вместимость одной команды |
| `LOCATION_CODE_*` | коды игровых локаций |

## Ограничение тестовой конфигурации

В `render.yaml` база SQLite размещается в `/tmp`. При полном пересоздании инстанса или новом деплое тестовые данные могут быть удалены. До регистрации реальных участников следует подключить постоянное хранилище или перенести базу в PostgreSQL.
