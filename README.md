# Telegram-бот «Последний хранитель»

Отдельный Telegram-бот иммерсивного культурно-просветительского проекта «Последний хранитель». Репозиторий и инфраструктура не связаны с ботом ОО «ЭРА».

## Суть механики

Дом Москвы в Ереване превращается в повреждённый Архив памяти России. Участники не листают справку и не проходят обычную викторину: они выполняют реальные задания, обсуждают последствия решений и постепенно открывают маршрут.

Основная формула:

**живая локация → цифровой код → игры команды → следующая локация → общий финал «Эффект бабочки».**

Даты программы: **16–17 ноября 2026 года**. Основная аудитория: молодые российские соотечественники 16–26 лет.

## Что изменено в версии 3.0

### Команду выдаёт только администратор

После регистрации участник получает статус «ожидает распределения». Цвет нельзя выбрать самостоятельно, и бот больше не распределяет новых участников автоматически.

Администратор открывает `/admin`, выбирает участника и выдаёт одну из пяти команд. Бот показывает текущее наполнение каждого цвета и не позволяет превысить вместимость.

### Следующая локация закрыта кодом

У каждой живой точки есть скрытый четырёхзначный код. Код автоматически создаётся в базе и доступен только администраторам через `/admin` → **«Коды живых локаций»**.

Участник может открыть только текущую точку своей команды. После правильного кода бот:

- отмечает этап пройденным для всей команды;
- открывает следующую точку по утверждённой схеме;
- открывает вторую цифровую игру этапа;
- предлагает сохранить командный выбор для финала.

Код другой локации, повторный код или попытка перескочить этап не работают.

### 50 разных игр

Для каждой команды подготовлено **10 собственных игр** — всего 50. На каждом из пяти живых этапов команда получает две цифровые миссии:

1. первая доступна при входе на текущий этап;
2. вторая открывается только после правильного кода живой локации.

Содержание связано с русским языком, литературой, театром, музыкой, кино, народными промыслами, историческими источниками, географией России и российской наукой. Вопросы команд не повторяются.

Полные варианты ответов выводятся в тексте сообщения. На кнопках остаются только короткие номера `1️⃣ 2️⃣ 3️⃣`, поэтому текст не обрезается на мобильных экранах.

## Маршруты из программы

Маршруты хранятся в отдельном файле `route_plan.json`, а не спрятаны внутри обработчиков бота.

| Команда | Последовательность |
|---|---|
| Красные | Код культуры → Цена открытия → Выбор эпохи → Голоса памяти → Открытые пространства |
| Белые | Цена открытия → Выбор эпохи → Голоса памяти → Открытые пространства → Код культуры |
| Оранжевые | Выбор эпохи → Голоса памяти → Открытые пространства → Код культуры → Цена открытия |
| Зелёные | Голоса памяти → Открытые пространства → Код культуры → Цена открытия → Выбор эпохи |
| Синие | Открытые пространства → Код культуры → Цена открытия → Выбор эпохи → Голоса памяти |

Временные блоки: `11:00–11:40`, `11:40–12:20`, `12:20–13:00`, `13:00–13:40`, `13:40–14:15`.

## Главное меню участника

- **📍 Текущая точка** — открытая сейчас живая локация и инструкция;
- **🎮 10 игр команды** — личный набор миссий и блокировки по маршруту;
- **🗺 Маршрут** — пять этапов со статусами `пройдено / открыто / закрыто`;
- **📜 Прогресс** — живые точки, игры и командные решения;
- **🗓 Программа** — программа дня, мастерские и открытые пространства;
- **❓ Архивариус** — обращение к организаторам.

Команда `/demo` сохраняет короткий экспертный деморежим без обязательной регистрации.

## Командные решения

После четырёх основных локаций участники выбирают один из двух вариантов. Первый подтверждённый выбор сохраняется как решение всей команды.

Решения меняют скрытые параметры Архива и формируют одну из финальных версий. Открытые пространства являются пятым маршрутным этапом, но не добавляют отдельного решения.

## Админ-панель

Основной экран `/admin` показывает:

- сколько человек зарегистрировано;
- сколько ждут команду;
- наполнение пяти команд;
- очередь ручного распределения;
- секретные четырёхзначные коды;
- прогресс каждой команды по маршруту и решениям.

Существующие служебные возможности бота — обращения, ответы, рассылки, финал, экспорт и управление администраторами — остаются в кодовой базе.

## Команды

- `/start` — регистрация или возвращение в игру;
- `/games` — десять игр своей команды;
- `/route` — маршрут и открытая точка;
- `/progress` — личный и командный прогресс;
- `/program` — программа проекта;
- `/demo` — экспертное демо;
- `/help` — правила прохождения;
- `/admin` — ручная выдача команд и контроль маршрута;
- `/whoami` — Telegram ID и роли;
- `/cancel` — отмена текущего действия.

## Структура данных

- `route_plan.json` — утверждённая схема пяти маршрутов;
- `game_catalog/*.json` — 10 игр для каждого цвета;
- `team_games.py` — загрузка и проверка каталога из 50 игр;
- `team_route_unlocks` — командное прохождение живых этапов;
- `team_game_progress` — индивидуальное прохождение цифровых игр;
- `live_location_codes` — скрытые четырёхзначные коды;
- `team_choices` — четыре командных решения для финала.

## Развёртывание на Render

1. Создайте Blueprint из репозитория.
2. Укажите секрет `BOT_TOKEN`.
3. Проверьте `SUPERADMIN_IDS`.
4. Дождитесь статуса `Live`.
5. Откройте `/health`: ожидается `status: ok` и username бота.
6. Отправьте `/start`.
7. Администратор открывает `/admin`, выдаёт тестовому участнику команду и смотрит коды локаций.

Webhook автоматически восстанавливается при каждом запуске.

## Локальный запуск

```bash
cp .env.example .env
# заполните BOT_TOKEN и при необходимости PUBLIC_BASE_URL
pip install -r requirements.txt
uvicorn main:web --reload
```

## Проверка

```bash
python -m compileall -q app.py admin_tools.py expert_ux.py main.py config.py game_data.py storage.py route_config.py team_games.py team_quest.py quest_common.py quest_route.py quest_games.py quest_admin.py
python -m unittest discover -s tests -v
```

Автотесты проверяют:

- 50 игр и отсутствие повторяющихся ID;
- ровно 10 игр на каждую команду;
- ровно две игры на каждую живую точку;
- соответствие порядка игр маршруту команды;
- наличие пяти уникальных четырёхзначных кодов;
- невозможность перейти от первого этапа ко второму без записи о правильном коде.

## Важное ограничение текущего хостинга

В бесплатной конфигурации Render SQLite находится в `/tmp`. При полном пересоздании сервиса или новом деплое регистрации, прогресс и созданные коды могут быть удалены. До открытия реальной регистрации необходимо подключить постоянный диск или PostgreSQL. После подключения постоянного хранилища коды сохраняются между перезапусками.
